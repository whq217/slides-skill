# 布局模式库

布局与主题完全解耦。主题控制"看起来怎样"（颜色/字体/质感），布局控制"怎么排列"（结构/比例/节奏）。共 33 种布局模式。

生成时根据**页面内容类型**自动匹配布局，同一主题、同类内容仍有多种布局可选。

---

## 使用规则

1. 每个 `<section>` 页面选择一种主布局
2. 主布局内的子区域可用辅助布局
3. **相邻页面避免使用相同布局** — 保持视觉节奏变化
4. 10页PPT至少使用5种不同布局
5. 布局选择依据：内容类型 > 数据量 > 信息密度

## 封面布局（3种）

### C-1 居中式
```
┌─────────────────────────────┐
│         装饰元素              │
│         标签                  │
│        大标题                 │
│        分割线                 │
│        副标题                 │
│        来源信息               │
└─────────────────────────────┘
```
- **两种变体：**
  - **标准居中**：内容垂直居中，几何装饰元素点缀，适用品牌发布、行业调研
  - **上下分栏**：上半部分放标题/副标题/来源，下半部分放主题色色块+关键数据预览，适用数据型报告、投资分析
    ```
    ┌─────────────────────────────┐
    │         标签                  │
    │        大标题                 │
    │        副标题                 │
    ├─────────────────────────────┤
    │   ████████ 色块+数据 ████████ │
    └─────────────────────────────┘
    ```

### C-2 卡片式
```
┌─────────────────────────────┐
│      ┌─────────────┐        │
│      │  标签        │        │
│      │  大标题      │        │
│      │  副标题      │        │
│      │  来源        │        │
│      └─────────────┘        │
└─────────────────────────────┘
```
- 内容集中在浮动卡片内
- 卡片有背景色/阴影/圆角
- 适用：精致报告、创意主题

### C-3 全出血 Hero
```
┌─────────────────────────────┐
│  ████████████████████████    │
│  ████████ 背景 ████████     │
│  ████████████████████████    │
│      标签                    │
│      大标题（白色叠加）       │
│      副标题                  │
│      CTA 按钮                │
└─────────────────────────────┘
```
- 全出血渐变/图片背景，`position: absolute` 铺满 slide
- 半透明遮罩层提升文字可读性
- 文字颜色强制白色 + `!important` 防御主题覆盖
- **三种内容排布变体：**
  - **底部左对齐**（默认）：文字块定位在左下象限 `left:0; top:50%; width:50%; height:50%`，象限内 `justify-content:center` 垂直居中，`align-items:flex-start` + `padding-left:60px` 内部左对齐。右上可放浮动数据指标
  - **居中偏下**：内容垂直水平居中，适合仪式感强的场景
  - **双色调分割**：左半深色+右半主题色，左侧标题+右侧关键数据
  - **左文右图**：左侧文字垂直居中，**强制 `text-align: left` + `align-items: flex-start`**（不随主题居中覆盖），右侧放置主题图/SVG/产品图，图片带交互效果（`z-index` 必须高于内容层以接收鼠标事件）：
    - **暗色系**：hover 发光（`drop-shadow` + `brightness(1.25)` + `translateY(-8px) scale(1.06)`），click 浮起（`translateY(-14px) scale(1.12)` + `brightness(1.4)` + 双层辉光加深）
    - **亮色系**：hover 主题色阴影（`box-shadow` 主题色 + `brightness(1.25)` + `translateY(-8px) scale(1.06)`），click 浮起（`translateY(-14px) scale(1.12)` + `brightness(1.4)` + 深阴影）
    - 三要素对齐内页 `.hl` 卡片：`transform` 浮起 + `filter: brightness` 提亮 + `shadow` 深度感，点击 toggle，再次点击恢复
    - 图片颜色必须随主题色板调整（SVG 替换 fill/stroke，位图用 `filter: hue-rotate` 或提供适配版本）
- 适用：数据型报告、投资分析、品牌发布、高端商务汇报
- 灵感：BMW 展厅式暗色 Hero、Apple 电影级全屏产品页

## 数据概览布局（4种）

### D-1 大色块+小数据条
```
┌─────────────┬───────┬───────┐
│   ████      │ 数据2  │ 数据3  │
│   ████ 数据1 │       │       │
│   ████      │       │       │
└─────────────┴───────┴───────┘
```
- 一个主数据占据1.3倍宽度，用主题色背景
- 其余数据用纯文字
- 不对称比例制造视觉层次

### D-2 等分卡片网格
```
┌───────┬───────┬───────┬───────┐
│ 数据1  │ 数据2  │ 数据3  │ 数据4  │
│ icon   │ icon   │ icon   │ icon   │
│ 标题   │ 标题   │ 标题   │ 标题   │
└───────┴───────┴───────┴───────┘
```
- 2/3/4等分
- 每个卡片结构相同：图标+标题+数值+说明
- 适用：4个以上并列数据

### D-3 指标条
```
┌─────────────────────────────────┐
│ 数据1      数据2      数据3      │
│ 大字号      大字号     大字号     │
│ 说明        说明       说明       │
├─────────────────────────────────┤
│ 补充说明区域                      │
└─────────────────────────────────┘
```
- 无边框/卡片，纯文字排列
- 数据用超大字号，紧凑排列
- 适用：简洁风格主题

### D-4 双行数据矩阵
```
┌───────┬───────┬───────┐
│ 数据1  │ 数据2  │ 数据3  │
├───────┼───────┼───────┤
│ 数据4  │ 数据5  │ 数据6  │
└───────┴───────┴───────┘
```
- 上下两行各3个数据
- 第二行可以是补充说明或次要指标
- 适用：6个数据的概览页

## 图文卡片布局（5种）

> **⚠️ 图片显示技术要点（必读）：**
> - 图片**必须使用 `background-image` 方案**，不使用 `<img>` 标签
> - 容器用 `<div class="bg-img" style="background-image:url(...)">`，CSS 用 `background-size:cover; background-position:center`
> - **禁止** `<img>` + `height:100%` + `object-fit:cover` 组合 — 在 Reveal.js transform 缩放环境下，flex/grid 推导的高度不是 CSS 规范中的"确定高度"，`height:100%` 和 `object-fit:cover` 无法可靠解析
> - 图片容器**必须设置 `position:relative`**（用于叠加文字覆盖层）
> - 用户本地图片用 base64 内联：`style="background-image:url('data:image/jpeg;base64,...')"`
> - **智能定位**：`.bg-img` 类已内置 `autoPositionImages()` JS 函数，会根据图片与容器的宽高比差异自动调整 `background-position`。竖图在宽容器中自动偏上显示，避免裁掉顶部重要内容。生成时无需手动设置 `background-position`，只需声明 `class="bg-img"` 即可

### IC-1 三栏图文卡片
```
┌──────────────┐  ┌──────────────┐  ┌──────────────┐
│ ████████████ │  │ ████████████ │  │ ████████████ │
│ ███ 图片 ███ │  │ ███ 图片 ███ │  │ ███ 图片 ███ │
│ ████████████ │  │ ████████████ │  │ ████████████ │
│              │  │              │  │              │
│  标题        │  │  标题        │  │  标题        │
│  描述文字    │  │  描述文字    │  │  描述文字    │
│  [标签]      │  │  [标签]      │  │  [标签]      │
└──────────────┘  └──────────────┘  └──────────────┘
```
- 三等分卡片，每张 = 图片区 55% + 文字区 45%
- 卡片使用 `display:flex; flex-direction:column`，图片区 `flex:0 0 55%`
- 悬停效果：上移 + 边框高亮 + 阴影加深
- 适用：产品线展示、团队介绍、三大特性
- 灵感：Apple 产品三栏展示、Stripe 功能卡片

### IC-2 居中大图 + 两侧文字

```
- 左侧 31% + 中间图片 38% + 右侧 31%
- 中间图片用 `.bg-img` 容器 + `border-radius:12px`
- 两侧各 3 个要点卡片，`justify-content:center` 垂直居中
- 适用：产品细节展示、技术架构中心辐射、核心功能拆解

- 底部可选数据指标条（4 个指标卡片横排）
- 适用：界面展示、案例分析、数据报告配图说明

- 适用：案例矩阵、产品矩阵、团队展示

## 特殊功能布局（5种）

### T-1 时间线/流程图
```
┌───────┬───────┬───────┐
│  ①    │──→ ②  │──→ ③  │
│ 步骤1  │ 步骤2  │ 步骤3  │
│ 描述   │ 描述   │ 描述   │
└───────┴───────┴───────┘
```
- 3-5个步骤水平排列，箭头或线段连接
- 每步：编号圆 + 标题 + 简短描述
- 适用：流程展示、项目阶段、开发路线图
- 灵感：Vercel "Develop → Preview → Ship" 三步工作流

┌─────────────────────────────┐
│  合作伙伴 / 信赖我们的企业    │
│  ┌───┐ ┌───┐ ┌───┐ ┌───┐   │
│  │ L │ │ L │ │ L │ │ L │   │

│  │ G │ │ G │ │ G │ │ G │   │

│  └───┘ └───┘ └───┘ └───┘   │
│  ┌───┐ ┌───┐ ┌───┐ ┌───┐   │
│  │ L │ │ L │ │ L │ │ L │   │
│  └───┘ └───┘ └───┘ └───┘   │

- 2行×4列或1行×6列的 Logo/品牌名网格
- Logo 用文字或图标代替（PPT 中通常用公司名+圆角方块）
- 适用：合作伙伴展示、客户案例、行业认可
- 灵感：Notion 信任背书条、Vercel 企业客户展示

### T-3 定价/对比表格

```
- 表格式对比，推荐方案用主题色背景突出
- 适用：方案对比、产品定价、功能对比
- 灵感：Stripe 定价页、Revolut 账户类型对比

### T-4 中心辐射 + 循环关系
```
              ┌───────┐
              │ 节点A  │
              │ icon   │
              └───┬───┘
                  │
    ← ─ ─ ─ ─ ─  ● 核心  ─ ─ ─ ─ ─ →
   ╱              │              ╲
┌───────┐    ┌─────────┐    ┌───────┐
│ 节点B  │ ←  │  中心概念 │  → │ 节点C  │
│ icon   │    └─────────┘    │ icon   │
└───────┘         ↓          └───────┘
   ↑          ┌─────────┐        │
   └──────────│  生态   │────────┘
              └─────────┘
```
- 中心核心圆 + 3-5个节点环绕排列（通常三角形或正多边形）
- 节点之间有循环箭头，表示双向/循环关系
- 节点不加卡片容器，只有图标 + 标题（适合内容极少、只有标题的场景）
- SVG 虚线或实线连接核心到各节点 + 节点间的循环路径
- 适用：商业模式三端关系、生态系统角色、多方协作模型
- 灵感：平台经济三方模型（用户-企业-管理端）、产业生态角色图

### T-5 环形/生态圈
```
         [外层1]  [外层2]
           ╲      ╱
   [外层6] ── (中A) ── (中B) ── [外层3]
             ╲  │  ╱
              (核心)
             ╱  │  ╲
   [外层5] ── (中D) ── (中C) ── [外层4]
           ╱      ╲
```
- 三层同心结构：核心圆 → 中间环（4个维度，圆形排列）→ 外层模块（6-8个）
- 中间环用圆形节点（带边框色圈 + 图标 + 标签），不用卡片
- 外层模块只用图标 + 标题，不加卡片容器
- 核心到外层用虚线辐射连接
- 适合内容极少、只需标题和图标的场景
- 适用：生态系统全景图、可持续发展模型、产业生态圈、多维能力模型
- 灵感：交通生态圈、数字经济生态图

## 布局选择流程

生成每个页面时，按以下步骤选择布局：

```
1. 确定内容类型
   封面 → C-1/C-2/C-3
   数据概览 → D-1/D-2/D-3/D-4
   图表页 → CH-1/CH-2/CH-3/CH-4
   要点列表 → L-1/L-2/L-3/L-4
   对比分析 → CP-1/CP-2/CP-3
   全出血/图文 → H-1/H-2
   图文卡片 → IC-1/IC-2/IC-3/IC-4A/IC-4B/IC-4C
   特殊功能 → T-1/T-2/T-3/T-4/T-5
   总结建议 → S-1/S-2/S-3

2. 检查已用布局，避免相邻页面重复

3. 在可选范围内随机选取一种

4. 将主题视觉风格叠加到选中布局上
```

## 主题与布局的叠加关系

主题只控制：
- 色块用什么颜色（`var(--accent)`）
- 文字用什么字体（`var(--font-display)`）
- 圆角大小、阴影风格
- 装饰元素的形状

布局控制：
- 区域怎么划分
- 用什么组件（卡片/色块/文字条/进度条）
- 主次信息的比例
- 空白与密集的节奏
